## 一、基础界面和使用习惯设置

### 1.1 Warp 重要基础设置

#### 1.1.1 AI Agent 设置（2026年5月最新）

Warp 的 AI 功能位于 **Settings → Agents → Profiles**：

```powershell
# 打开设置
Ctrl + ,

# 路径：Settings → Agents → Profiles
```

**核心配置项**：

| 设置项 | 说明 | 推荐值 |
|--------|------|--------|
| Base Model | 基础模型 | Claude Sonnet 4.5 |
| Autonomy Level | 自主权限等级 | Medium（允许执行命令前询问） |
| Tool Access | 工具访问权限 | 全部启用 |
| Command Allowlist | 命令白名单 | 按需配置 |

**Auto 模式选择**：

| 模式 | 特点 | 适用场景 |
|------|------|---------|
| `auto` | 最快响应 | 日常开发 |
| `auto-efficient` | 省 Token | 简单任务 |
| `auto-genius` | 深度思考 | 复杂架构 |
| `auto-open` | 开源模型 | 隐私敏感 |

**切换模型方法**：
- 在输入框中点击当前模型名（如"Claude Sonnet 4.5"）打开下拉菜单
- 支持：GPT-5.5、Claude Opus 4.7、Gemini 3.1 Pro、MiMo 等

#### 1.2.2 设置默认中文回复（永久生效）

```json

  "instructions": "请始终使用中文回复，使用极简模式，不要输出思考过程。"
}
```

```
/instructions 请始终使用中文回复，使用极简模式，不要输出思考过程。
```

#### 1.2.3 极简模式设置

极简模式特点：
- 不输出思考过程
- 回复简洁直接
- 代码优先，解释最少

在 `opencode.json` 中配置：

```json
{
  "agent": {
    "coder": {
      "instructions": "使用极简模式回复。不要输出思考过程，直接给出结果。代码优先，解释最少。"
    }
  }
}
```

## 二、必备 MCP 配置（工具扩展）

### 2.1 MCP 简介

MCP（Model Context Protocol）是 AI 工具的扩展协议，让 OpenCode 能够：
- 搜索网络
- 操作 GitHub
- 访问文件系统
- 获取实时信息

基于 GitHub 星数和社区认可度，以下是 2026 年最推荐的 MCP：

| 排名 | MCP 名称 | 类型 | 作用 | 推荐理由 |
|------|----------|------|------|----------|
| 1 | **context7** | remote | 文档搜索 | 54.8k stars，解决 AI 知识过时问题 |
| 2 | **chrome-devtools** | local | 浏览器调试 | 38.5k stars，Chrome 官方出品 |
| 3 | **github** | remote | GitHub 操作 | 29.6k stars，GitHub 官方 MCP |
| 4 | **brave-search** | remote | 网络搜索 | 隐私优先的搜索引擎 |
| 5 | **gh_grep** | remote | 代码搜索 | 快速搜索 GitHub 代码 |
| 6 | **git** | local | Git 操作 | 提交、分支、状态查询 |
| 7 | **filesystem** | local | 文件系统 | 增强文件操作能力 |
| 8 | **memory** | local | 知识图谱 | 长期记忆存储 |

将以下内容配置到 `opencode.json` 的 `mcp` 部分：

      "url": "https://mcp.context7.com/mcp",
      "enabled": true

      "type": "local",
      "command": ["npx", "-y", "@anthropic-ai/chrome-devtools-mcp"],
      "enabled": true
    },
    "github": {
      "type": "remote",
      "url": "https://api.githubcopilot.com/mcp",
      "oauth": {},
      "enabled": true
    },
    "brave-search": {
      "type": "remote",
      "url": "https://mcp.brave.com/mcp",
      "oauth": {},
      "enabled": true
    },
    "gh_grep": {
      "type": "remote",
      "url": "https://mcp.grep.app",
      "enabled": true
    },
    "git": {
      "type": "local",
      "command": ["npx", "-y", "mcp-server-git"],
      "enabled": true
    },
    "filesystem": {
      "type": "local",
      "command": ["npx", "-y", "@modelcontextprotocol/server-filesystem"],
      "args": ["D:\\Xcode"],
      "enabled": true
    },
    "memory": {
      "type": "local",
      "command": ["npx", "-y", "@modelcontextprotocol/server-memory"],
      "enabled": true
    },
    "sentry": {
      "type": "remote",
      "url": "https://mcp.sentry.dev/mcp",

### 4.1 MiMo 模型选择

| 模型 | 特点 | 适用场景 |
|------|------|---------|
| MiMo-V2.5-Pro | 更强、更准 | 复杂任务、代码生成 |
| MiMo-V2.5 | 更快、更省 | 简单问答、日常对话 |

### 4.2 合理使用模型

```json

  "agent": {
    "coder": {
      "model": "xiaomi-mimo/MiMo-V2.5-Pro"
    },
    "tester": {
      "model": "xiaomi-mimo/MiMo-V2.5"
    }
  }
}
```

### 4.3 会话管理：压缩 vs 新建（量化指标）

#### 核心区别

| 操作 | 效果 | 适用场景 |
|------|------|---------|
| `/compact` | 压缩当前会话，保留关键信息 | 同一任务内，上下文膨胀 |
| `/new` | 新建会话，完全清空上下文 | 任务切换，或上下文已无法压缩 |

#### 何时压缩（`/compact`）

| 指标 | 阈值 | 说明 |
|------|------|------|
| 对话轮数 | **>15-20 轮** | 同一任务内的对话开始变慢 |
| Token 使用量 | **>模型上下文窗口的 70%** | 如 MiMo 128K 窗口，超过 90K 时 |
| 工具调用次数 | **>30 次** | 工具输出累积过多 |
| 性能下降 | AI 回答质量明显下降 | 遗漏之前的讨论内容 |

#### 何时新建（`/new`）

| 场景 | 建议 |
|------|------|
| **任务切换** | 不同功能模块/Issue → 新建会话 |
| **上下文污染** | 包含大量无关调试输出/错误日志 → 新建 |
| **长时间运行** | **>45-60 分钟**的持续交互 → 新建 |
| **Token 接近上限** | 上下文窗口接近满载 → 新建 |
| **计划完成** | 一个完整的 plan 执行完毕 → 新建 |

#### 操作命令

```powershell
# 压缩上下文
/compact

# 新建会话
/new
# 或
Ctrl + N
```

### 6.3 常见问题预防与解决

#### 6.3.1 Token 消耗过快

**预防**：
- 定期新建会话
- 使用 /compact 压缩上下文
- 精准提问，减少无关对话

**解决**：
```
/compact
```

#### 6.3.2 响应变慢

**预防**：
- 保持会话精简
- 避免超长上下文
- 及时关闭不用的会话

**解决**：
```
/new  # 新建会话
```

#### 6.3.3 回答不准确

**预防**：
- 提供充足上下文
- 使用文件引用
- 明确指定需求

**解决**：
```
# 重新描述需求
"我需要的是在 src/lib/parsers/ 目录下添加小红书解析器"
```

#### 6.3.4 MCP 工具不工作

**预防**：
- 检查 MCP 配置
- 验证网络连接
- 确认 OAuth 授权

**解决**：
```powershell
# 检查 MCP 状态
opencode mcp list

# 重新授权
/connect
```

#### 6.3.5 代码质量问题

**预防**：
- 使用 code-reviewer Skill
- 定期运行 lint
- 编写单元测试

**解决**：
```powershell
# 运行 lint
npm run lint

# 运行测试
npm run test:run
```
